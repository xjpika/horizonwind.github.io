APIO2019 路灯

2020-06-22

看到点对，什么思路？转化为平面问题。

显然点对连通性可以转化到平面上。

设点对(x,y)(x,y)(x,y)在平面上表示为坐标(x,y)(x,y)(x,y)。

就是把点对映射到坐标上。

我们用 set 来维护亮灯的连续段。

设当前时间为ttt，总时间为TTT。

对于一个点xxx,设xxx可以到达最左边的位置为lblblb，x+1x+1x+1可到达最右边的位置为rbrbrb。

修改

如果[x,x+1][x,x+1][x,x+1]之间的路灯开了，那么显然[lb,x][\text{lb},x][lb,x]内的点与[x+1,rb][x+1,\text{rb}][x+1,rb]内的点就可以连通了！

[lb,x],[x+1,rb][\text{lb},x],[x+1,\text{rb}][lb,x],[x+1,rb]连通，我们转化为矩形加T−tT-tT−t，表示这个矩形在ttt时刻亮了。

这个矩形左下端点为[lb,x+1][\text{lb},x+1][lb,x+1]，右上端点为[x,rb][x,\text{rb}][x,rb]。

如果[x,x+1][x,x+1][x,x+1]之间的路灯断了，[lb,x][\text{lb},x][lb,x]与[x+1,rb][x+1,\text{rb}][x+1,rb]不再连通，我们转化为矩形减T−tT-tT−t。

表示这个矩形在ttt时刻又暗了。

查询

发现询问就是矩形单点查询。

假设这个位置在t1t_1t1​时连通，t2t_2t2​时断开，

根据刚才的修改操作，这个点对连通的时间为T−t1−(T−t2)=t2−t1T-t_1-(T-t_2)=t_2-t_1T−t1​−(T−t2​)=t2​−t1​

正好相等！

注意：如果查询的两点依然连通，答案还需要减去(T−t)(T-t)(T−t)

我们成功把问题转化为了普通的矩形加，单点查问题。

这就是普普通通的二维数点问题。

由于有时间轴，所以等价于三维偏序问题。

所以就有两种解法：

算法111：CDQ分治+扫描线。

很显然这个可以用 CDQ 分治维护。

矩形加不好处理，先用扫描线把矩形差分成444个操作（和[HNOI2015]接水果\texttt{[HNOI2015]接水果}[HNOI2015]接水果 很像）。

第一维时间排序掉，第二维xxx坐标 CDQ ，第三维yyy坐标用差分+树状数组。

算法222：树套树+扫描线。

用树套树可以支持在线。

先把矩形加用扫描线差分成444个操作。

然后就是普普通通的单点修改，区间查询了。

这个不就是树套树可以维护的东西吗？

树状数组维护xxx坐标，动态开点线段树维护yyy坐标即可。

总结

两个算法复杂度都是O(nlog⁡22n)O(n\log_2^2n)O(nlog22​n)的。

而且有树状数组在，常数巨小。可以通过此题。

int n,N,T,root[600001],cnt;
struct node{
	int l,r,sum;
}seg[15020001];
#define mid ((lb+rb)>>1)
void Update(int &rt,int lb,int rb,int pos,int w){
	if (!rt) rt=++cnt;seg[rt].sum+=w;
	if (lb==rb) return;
	(mid>=pos?Update(seg[rt].l,lb,mid,pos,w):Update(seg[rt].r,mid+1,rb,pos,w));
}
int Query(int rt,int lb,int rb,int l,int r){
	if (!rt||lb>r||rb<l) return 0;
	if (lb>=l&&rb<=r) return seg[rt].sum;
	return Query(seg[rt].l,lb,mid,l,r)+Query(seg[rt].r,mid+1,rb,l,r);
}
inline void update(int pos,int r,int w){
	for (;pos<=n+1;pos+=pos&-pos) Update(root[pos],1,n+1,r,w);
}
inline int query(int posx,int posy){
	int ans=0;
	for (;posx;posx-=posx&-posx) ans+=Query(root[posx],1,n+1,1,posy);
	return ans;
}
inline void Add(int x1,int y1,int x2,int y2,int W){
	update(x1,y1,W);update(x1,y2+1,-W);
	update(x2+1,y1,-W);update(x2+1,y2+1,W);
}
